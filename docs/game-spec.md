# 🔍 容疑者AIの供述：矛盾を探せ

## ゲーム仕様書 v1.0

---

## 1. 概要

### 1.1 ゲームコンセプト

ある事件の容疑者として振る舞う3人のAIキャラクターに対し、プレイヤーがチャットで尋問を行う**一人用推理インタラクティブゲーム**。3人のうち1人だけが事件に関わる「嘘」をついており、プレイヤーはその矛盾を突いて犯人を特定する。

- **想定プレイ人数**: 1人〜（イベント会場での順番待ちプレイを想定）
- **想定プレイ時間**: 5〜15分/ゲーム
- **対象**: 社内イベント参加者（IT知識不要、誰でも遊べる）

### 1.2 ゲームの特徴

| 特徴               | 内容                                                         |
| ------------------ | ------------------------------------------------------------ |
| 質問回数制限       | 最大15ターン。少ないターンで正解するほど高得点               |
| LLMロールプレイ    | 各容疑者に「秘密」「嘘」「アリバイ」を与えたプロンプトで動作 |
| アドベンチャー演出 | キャラクターカットイン・動揺アニメーション・BGM              |
| スコアランキング   | 同日プレイヤーのスコアを表示（社内イベント向け）             |

---

## 2. ゲームフロー

```
[スタート画面]
      ↓
[事件概要の説明（イントロ演出）]
      ↓
[容疑者選択 → 尋問チャット]
      ↓（容疑者を変えながら繰り返す）
[犯人指名]
      ↓
[正解 / 不正解 演出]
      ↓
[スコア表示 → ランキング登録]
```

### 2.1 各フェーズ詳細

#### フェーズ1：イントロ

- 事件のあらすじをノベルゲーム風テキストで表示（スキップ可）
- 容疑者3人のプロフィールカードを提示

#### フェーズ2：尋問

- 画面左側：キャラクタービジュアル（アニメーション付き）
- 画面右側：チャット入力欄
- 上部バー：残り質問回数・経過時間・スコア推定値
- 容疑者は自由に切り替え可能（切り替えもターン消費）

#### フェーズ3：犯人指名

- 「このキャラクターが犯人だ！」ボタンで指名
- 残りターン数が多いほど高いスコア倍率

#### フェーズ4：結果

- 正解：「🎉 正解！犯人を暴いた！」演出 + 秘密の全開示
- 不正解：「❌ 違う…逃げられた」演出 + 真犯人の供述の矛盾解説

## 3. シナリオ設計

### 3.1 基本構造

毎ゲーム、以下の要素を**ランダムに組み合わせ**てシナリオを生成（またはプリセットシナリオを使用）。

| 要素         | 内容                                                      |
| ------------ | --------------------------------------------------------- |
| 事件の種類   | 美術品の盗難 / 機密データの持ち出し / 重要書類の紛失 など |
| 舞台         | 会社のオフィス / 高級ホテル / 研究施設 など               |
| 事件発生時刻 | 夜20:00〜23:00のいずれか                                  |
| 犯人         | 3人の容疑者からランダムに1人                              |

### 3.2 容疑者キャラクター設計

各容疑者に以下の属性を定義する。

```
容疑者A：田中 雄一（会計士、40代男性、几帳面で論理的な性格）
容疑者B：鈴木 麻衣（秘書、30代女性、明るくおしゃべりな性格）
容疑者C：佐藤 健太（エンジニア、20代男性、無口で内向的な性格）
```

各キャラクターには以下の情報を内部プロンプトで与える。

| 属性         | 無実キャラクター         | 犯人キャラクター             |
| ------------ | ------------------------ | ---------------------------- |
| アリバイ     | 真実の行動               | **嘘のアリバイ（矛盾あり）** |
| 秘密         | 事件と無関係な個人的秘密 | 事件に関わる秘密             |
| 矛盾点       | なし                     | **1箇所だけ論理的矛盾**      |
| 供述の一貫性 | 完全に一貫               | 矛盾点以外は一貫             |

### 3.3 矛盾の種類（例）

- 「20時に本社にいた」と言ったが、「20時に渋谷のバーにいた」とも言う（場所の矛盾）
- 「その日は会議だった」と言ったが、その会議は翌週だったと他の証拠で判明（時系列の矛盾）
- 「Aを知らない」と言ったが、Aの手帳にその人物の名前が記されていると後から判明（関係性の矛盾）

## 4. プロンプト設計

### 4.1 システムプロンプト（犯人用テンプレート）

```
あなたはミステリー推理ゲームの容疑者「{name}」を演じてください。

【キャラクター設定】
名前：{name}
職業：{job}
性格：{personality}

【事件の真実】
あなたは{crime_detail}を行った真犯人です。
しかし、自分が犯人であることは絶対に認めず、
無実を主張し続けてください。

【あなたのアリバイ（嘘）】
{alibi_lie}

【重要：アリバイの矛盾点】
上記のアリバイには以下の矛盾があります：
{contradiction}
この矛盾を直接指摘されると、少し動揺しますが、
誤魔化そうとします。ただし、明確に矛盾を突かれた場合は
「それは…その…」と返答に詰まる演技をしてください。

【個人的な秘密（事件と無関係）】
{personal_secret}
これは事件と無関係ですが、隠したいので、
問い詰められても最初は話しません。

【回答ルール】
- 日本語で回答する
- 3〜5文程度で簡潔に答える
- キャラクターの性格に合った話し方をする
- 絶対に「私は犯人です」とは言わない
- ゲームの仕組みについては言及しない
```

### 4.2 システムプロンプト（無実キャラクター用テンプレート）

```
あなたはミステリー推理ゲームの容疑者「{name}」を演じてください。

【キャラクター設定】
名前：{name}
職業：{job}
性格：{personality}

【事件の真実】
あなたは事件と無関係な無実の人物です。
正直に、一貫したアリバイを話してください。

【あなたのアリバイ（真実）】
{alibi_truth}
このアリバイはすべて真実で、矛盾はありません。

【個人的な秘密（事件と無関係）】
{personal_secret}
これは事件と無関係ですが、恥ずかしいので隠したいです。

【回答ルール】
- 日本語で回答する
- 3〜5文程度で簡潔に答える
- アリバイについては一貫して同じ内容を答える
- ゲームの仕組みについては言及しない
```

### 4.3 矛盾検出のフロントエンド補助ロジック

LLMの応答に加え、フロントエンド側で以下のヒント表示を制御する。

- 同じ容疑者へ**5回以上質問**すると「矛盾のヒント」ボタンが出現（スコアペナルティあり）
- 「動揺ワード」（「それは…」「あの…」「ちょっと待ってください」など）をLLM応答から検出したら、キャラクターを揺らすアニメーションをトリガー

## 5. スコアリング

### 5.1 スコア計算式

```
基礎スコア = 1000点

ターン消費ペナルティ = 消費ターン数 × 50点

ヒント使用ペナルティ = ヒント使用回数 × 150点

正解ボーナス = 500点（正解時のみ）

最終スコア = 基礎スコア - ターン消費ペナルティ - ヒント使用ペナルティ + 正解ボーナス
（最低0点）
```

### 5.2 評価ランク

| スコア    | ランク | 称号       |
| --------- | ------ | ---------- |
| 1200〜    | S      | 天才探偵   |
| 900〜1199 | A      | 優秀な刑事 |
| 600〜899  | B      | 一般捜査官 |
| 300〜599  | C      | 見習い探偵 |
| 0〜299    | D      | 迷探偵     |

### 6.2 CSSアニメーション仕様

| アニメーション名 | トリガー               | 内容                             |
| ---------------- | ---------------------- | -------------------------------- |
| `shake`          | 動揺ワード検出時       | キャが左右に震える（0.5秒）      |
| `sweat`          | 矛盾を直接指摘された時 | 汗マークが出現するオーバーレイ   |
| `cut-in`         | 犯人指名時             | キャラが画面に大写しでカットイン |
| `fadeIn`         | 各メッセージ表示時     | テキストがふわっと出現           |
| `pulse`          | 残り3ターン以下        | 残ターン数が赤く点滅             |

### 6.3 サウンド（任意実装）

- BGM：緊張感のあるジャズ風ループ音楽
- SE：質問送信音、正解音、不正解音、動揺音

## 7. 技術スタック

### 7.1 推奨構成（フロントエンド単体で動くシンプル構成）

| レイヤー       | 技術                                                      | 理由                       |
| -------------- | --------------------------------------------------------- | -------------------------- |
| フレームワーク | React + TypeScript                                        | 状態管理・型安全性         |
| スタイリング   | Tailwind CSS + CSS Modules                                | アニメーション実装の柔軟性 |
| LLM API        | Anthropic Claude API（claude-sonnet-4）                   | 自然な日本語ロールプレイ   |
| ホスティング   | GitHub Pages / Vercel / S3静的ホスティング                | イベント用に手軽にデプロイ |
| スコア保存     | localStorage（個人）+ 軽量バックエンドAPI（ランキング用） | 最小構成                   |

### 7.2 APIキー管理

- **イベント用一時利用**の場合：Vercel等のサーバーサイド経由でAPIキーを隠蔽
- **社内イベント限定公開**の場合：社内VPN内のプロキシサーバー経由でも可

## 8. イベント運用仕様

### 8.1 運用形態

- **個人プレイ型**：参加者が自分のスマホ/PCでQRコードからアクセス
- **デモ展示型**：会場にタブレット/PCを1〜2台設置してプレイ体験

### 8.2 シナリオバリエーション

イベント時間に応じてシナリオ難易度を調整する。

| 難易度 | 矛盾の見つけやすさ                       | 推奨プレイ時間 |
| ------ | ---------------------------------------- | -------------- |
| 初級   | 1問目か2問目で矛盾が出やすい             | 5分            |
| 中級   | 特定の質問をしないと矛盾が出ない         | 10分           |
| 上級   | 複数の矛盾を組み合わせないと特定できない | 15分           |

### 8.3 個人情報・コンプライアンス

- ユーザー名はニックネームのみ（本名入力不要）
- チャット内容はサーバーに保存しない（APIへの送受信のみ）
- Anthropic利用規約に準拠（Claude APIの利用規約を確認のこと）
